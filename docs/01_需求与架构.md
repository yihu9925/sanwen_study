# 需求分析与整体架构设计

## 环境适配说明

> 本设计基于 **HarmonyOS 6.0.2 SDK / OpenHarmony API 20** 开发。
> - 导入方式：统一使用 `@kit.*` 命名空间（如 `@kit.ArkData`、`@kit.ArkUI`、`@kit.AbilityKit`）
> - 严格模式：禁止 `any`、`as any`、`Record<string, any>`。使用 `object` 或自定义 interface
> - 状态管理：`@StorageLink` / `@LocalStorageLink` 须配合显式类型标注
> - Canvas2D：API 20 增强了 CanvasRenderingContext2D 的绘制性能

## 产品定位与核心价值

### 产品定位
"三问高效学习机"是一款以用户提问为核心触发机制的**智能学习工具**。

- 核心机制：提问 → 课程生成 → 三问学习法 → 深度理解
- 目标用户：大学生、知识工作者、自学者
- 学习范围：任何大学课程或领域知识
- 学习方式：柔性、自主、个性化

### 三大核心价值

| 价值 | 说明 |
|-----|------|
| **按需学习** | 课程内容完全由用户提问触发，避免信息过载 |
| **认知加速** | 通过三问法快速建立框架、挖掘深层、验证理解 |
| **个性化适配** | AI资料 + 用户资料融合，形成专属知识库 |

## 核心业务流程

### 用户学习全生命周期

```
1️⃣ 提问阶段
   用户在搜索框输入学习问题
   ↓
   系统自动识别领域、生成课程框架
   ↓
   创建课程卡片、进入学习空间

2️⃣ 三问学习阶段
   第一问：知识图谱呈现
   ├─ 展示学科底层逻辑框架
   │
   第二问：争议点分析
   ├─ 对比多源观点、挖掘深层分歧
   │
   第三问：认知测评
   ├─ 6个维度自测、评估理解深度

3️⃣ 完成与反馈阶段
   生成学习报告、能力雷达图、测评分析
   ↓
   支持数据导出、课程归档、经验分享
```

## 核心功能模块详解

### 模块1: 柔性课程生成

**触发机制**
- 用户提问 → LLM识别领域 → 自动创建课程 → 进入学习空间

**课程生命周期状态**

| 状态 | 描述 | UI表现 | 数据库值 |
|-----|------|--------|---------|
| **created** | 首次提问，框架已生成 | 卡片显示，进度0% | 'created' |
| **active** | 正在学习中 | 卡片高亮，进度条更新 | 'active' |
| **completed** | 三问全部完成 | 卡片置灰，显示"已完成" | 'completed' |
| **archived** | 用户归档 | 移至归档列表，不在首页显示 | 'archived' |

**多课程并行支持**
- 用户可创建多个课程（跨学科）
- 课程按最近访问时间排序
- 未激活课程默认隐藏（保持界面简洁）

### 模块2: 复合知识库构建

**三源数据融合**
```
AI智能资料补充
    ├─ 联网搜索权威论文
    ├─ 官方文档
    └─ 行业报告

+ 用户上传资料
    ├─ PDF
    ├─ Word
    └─ Markdown
    
= 复合知识库
    ├─ 语义融合
    ├─ 去重与排序
    └─ 智能推荐
```

**资料处理流程**
1. 自动解析格式（使用 `@ohos.file.fs` 读取文件，提取文本）
2. 语义向量化（可选，用于相似度计算）
3. 关键词索引（用于快速搜索）
4. 分类归档（按学科、按来源）

### 模块3: 三问认知引擎

#### 第一问：核心心智模型提取
- **输入**：复合知识库的所有文档
- **处理**：LLM分析，提取学科底层逻辑框架
- **输出**：知识图谱（节点=核心概念，连线=逻辑关系）
- **交互**：
  - 点击节点展开/折叠详情
  - 查看关联资料
  - 搜索知识点高亮

#### 第二问：学术分歧挖掘
- **输入**：多源资料观点对比
- **处理**：LLM识别关键争议点、整理正反方证据
- **输出**：争议点卡片（正方观点 vs 反方观点）
- **交互**：
  - 切换不同争议主题
  - 查看详细证据
  - AI自动总结主流观点

#### 第三问：深度测评生成（自由答题模式）

> ⚠ **设计原则**（来源于需求文档明确要求）：
> - "自己回答自己产品的学习提问"
> - "产品应该对学员的学习效果提出评价和对话原始记录"
> - "不要拿AI生成答案喂AI，这样会扣分"
> 
> 因此第三问采用**开放题模式**，而非选择题。系统出题后，用户自由撰写回答，LLM评判回答质量。

- **基础**：布鲁姆认知层级（6个维度）
- **维度**：记忆 → 理解 → 应用 → 分析 → 评价 → 创造
- **题量**：每个维度2-3道开放题（共12-18题）
- **交互流程**：
  1. 系统展示题目（含难度、布鲁姆层级标签）
  2. 用户在文本框中自由输入回答（非选择题）
  3. 调用LLM评判用户回答质量：给出评分（0-100）+ 针对性评语
  4. 保存完整的**对话原始记录**（用户回答原文 + LLM评语 + 时间戳）
  5. 全部答题完毕后生成评估报告
- **报告内容**：
  - 每道题：用户回答原文、AI评语、评分
  - 按布鲁姆维度汇总得分率
  - 雷达图可视化6维度能力
  - 学习建议（基于薄弱维度生成）
- **功能**：
  - 自由输入回答（多行文本框）  ← 替代原来的选项选择
  - LLM智能评分与解析
  - 对话记录保存与回溯
  - 错题薄弱处识别（基于评分<60分的题目）
  - 综合学习报告生成

### 模块4: 学习进度追踪

**可视化元素**
1. **进度条**：整体完成度百分比
2. **能力雷达图**：6维度能力评分展示
3. **学习时长统计**：周/月数据展示
4. **错题分析**：薄弱知识点识别

**个性化提醒**
- 任务提醒：某课程已7天未学
- 深度提示：争议点解读推送
- 复习建议：错题集自动复习计划

### 模块5: 用户交互与社区

**卡片操作**
- 长按菜单：上传资料、导出、归档、删除
- 划动删除：快速删除课程
- 拖拽排序：自定义课程顺序

**数据导出**
- PDF：学习报告、知识图谱图片
- JSON：完整课程数据、测评答卷
- Markdown：知识库导出、笔记整理

## ArkTS 严格模式注意事项 (API 20)

1. **禁止 any**：所有变量必须有具体类型
2. **禁止 Record<string, any>**：使用 `Record<string, string>` 或自定义 interface
3. **禁止 as any 强制转换**：使用 as 仅用于类型缩窄
4. **状态变量**：`@State`、`@Prop`、`@Link` 必须标注类型，不可省略
5. **Map 类型**：不能直接用于 @State，改用数组或 Record
6. **资源引用**：使用 `$r('app.string.xxx')` 而非字符串硬编码

### 场景1: 创建课程 — 完整数据流

```
用户:  在HomePage搜索框输入"什么是深度学习？" → 点击搜索

① UI事件层:
   HomePage.searchText 更新
   → SearchBox.onSubmit() 触发
   → 设置 isLoading = true (显示Loading动画)
   → 调用 CourseService.createCourse(searchText)

② Service逻辑层:
   CourseService.createCourse():
     ├─ LLMService.identifyDomain("什么是深度学习？")
     │    → fetch POST /v1/chat/completions
     │    → 返回 "人工智能"
     │
     ├─ UUID生成 courseId
     ├─ INSERT INTO course (id, user_id, title, domain, ...)
     │    → RelationalStoreManager.insert()
     │    → 返回 rowId
     ├─ INSERT INTO knowledgebase (id, course_id, ...)
     │
     ├─ 同步返回 courseId 给UI
     │
     └─ 后台异步执行 (不阻塞UI):
          ├─ LLMService.generateKnowledgeGraph()
          │    → 返回 GraphNode[]
          │    → UPDATE knowledgebase SET knowledge_graph = ?
          │    → 发送事件 'graph_ready' (见EventBus)
          │
          ├─ LLMService.discoverControversies()
          │    → 返回 Controversy[]
          │    → UPDATE knowledgebase SET controversies = ?
          │    → 发送事件 'controversy_ready'
          │
          └─ LLMService.generateQuestions()
               → 返回 Question[]
               → 循环 INSERT INTO question
               → 发送事件 'questions_ready'

③ UI更新层:
   HomePage:
     └─ await CourseService.createCourse() 返回 courseId
     └─ isLoading = false
     └─ router.pushUrl → 跳转 CourseDetailPage

   CourseDetailPage (在 aboutToAppear 中):
     └─ 监听 EventBus 'graph_ready' → 重新加载知识图谱
     └─ 监听 EventBus 'controversy_ready' → 重新加载争议点
     └─ 监听 EventBus 'questions_ready' → 重新加载题目
```

## 全局事件总线 (EventBus)

> 用于Service层完成异步任务后通知UI层刷新

```typescript
// services/EventBus.ts
type EventHandler = (data?: object) => void;

export class EventBus {
  private static handlers: Record<string, EventHandler[]> = {};

  // 订阅事件
  static on(event: string, handler: EventHandler): void {
    if (!this.handlers[event]) {
      this.handlers[event] = [];
    }
    this.handlers[event].push(handler);
  }

  // 取消订阅
  static off(event: string, handler: EventHandler): void {
    const list = this.handlers[event];
    if (list) {
      this.handlers[event] = list.filter(h => h !== handler);
    }
  }

  // 触发事件
  static emit(event: string, data?: object): void {
    const list = this.handlers[event];
    if (list) {
      for (const handler of list) {
        handler(data);
      }
    }
  }

  // 清空所有订阅
  static clear(): void {
    this.handlers = {};
  }
}
```

### 事件列表

| 事件名 | 触发时机 | 载荷数据 | 监听方 |
|-------|---------|---------|--------|
| `graph_ready` | LLM生成知识图谱完成 | `{ courseId, graphNodes }` | CourseDetailPage Tab1 |
| `graph_updated` | 重新生成图谱完成 | `{ courseId, graphNodes }` | CourseDetailPage Tab1 |
| `controversy_ready` | LLM挖掘争议点完成 | `{ courseId, controversies }` | CourseDetailPage Tab2 |
| `questions_ready` | LLM生成题目完成 | `{ courseId, questionCount }` | CourseDetailPage Tab3 |
| `progress_updated` | 答题提交后进度更新 | `{ courseId, progressPercent, radarData }` | CourseDetailPage, HomePage |
| `course_created` | 新课程创建完成 | `{ courseId, title }` | HomePage |
| `document_added` | 文件上传完成 | `{ courseId, docTitle }` | CourseDetailPage |
| `theme_changed` | 主题模式切换 | `{ themeMode }` | 全局 |

### EventBus 使用示例

```typescript
// 在 CourseDetailPage 中监听异步结果
@Component
struct CourseDetailPage {
  aboutToAppear(): void {
    EventBus.on('graph_ready', (data) => {
      if ((data as Record<string, string>)?.courseId === this.courseId) {
        this.loadGraph();
      }
    });

    EventBus.on('progress_updated', (data) => {
      if ((data as Record<string, string>)?.courseId === this.courseId) {
        this.progressPercent = Number((data as Record<string, number>)?.progressPercent) || 0;
      }
    });
  }

  aboutToDisappear(): void {
    // 离开页面时取消订阅，防止内存泄漏
    EventBus.clear();
  }
}

// 在 Service 中触发事件
// LLMService.generateKnowledgeGraph() 完成后
EventBus.emit('graph_ready', {
  courseId: courseId,
  graphNodes: graphData
});
```

struct SkeletonCard {

          Rect().width('60%').height(14).fill('#E5E7EB').borderRadius(4)

        }

    }

    .borderRadius(12)

// 在页面中使用
@Component
struct HomePage {
  @State pageState: PageState = PageState.LOADING;

  build() {
    Column() {
      if (this.pageState === PageState.LOADING) {
        // 显示骨架屏
        ForEach(Array(4).fill(0), () => { SkeletonCard() });
      } else if (this.pageState === PageState.EMPTY) {
        // 空状态引导

      } else if (this.pageState === PageState.ERROR) {
        // 错误态

      } else {
        // 正常内容
        CourseList({ courses: this.courses });
      }
    }
  }
}

## 数据一致性保障

### 乐观更新 (Optimistic Update)

对于非关键操作（如点击完成第一问），先更新UI再写DB：

```typescript
// 1. 先更新UI（即时反馈）
this.progressPercent = Math.min(100, this.progressPercent + 33);

// 2. 再写DB（异步，不阻塞UI）
CourseService.updateProgress(this.courseId, 1).catch(() => {
  // 3. 写DB失败 → 回滚UI
  this.progressPercent = Math.max(0, this.progressPercent - 33);
  promptAction.showToast({ message: '保存失败，请重试' });
});
```

### 后台任务轮询

LLM生成是耗时的异步任务，页面通过轮询或EventBus获取结果：

```typescript
// 方案A: EventBus（推荐）
EventBus.on('graph_ready', (data) => {
  if (data?.courseId === this.courseId) {
    this.loadGraph();
  }
});

// 方案B: 轮询（兜底）
startPolling(): void {
  this.pollTimer = setInterval(async () => {
    const graph = await KnowledgeBaseService.getKnowledgeGraph(this.courseId);
    if (graph.length > 0) {
      this.graphData = graph;
      clearInterval(this.pollTimer);
    }
  }, 3000); // 每3秒查一次
}
```
