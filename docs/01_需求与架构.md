# 需求分析与整体架构设计

## 环境适配说明

> 本设计基于 **HarmonyOS 6.0.2 SDK / OpenHarmony API 20** 开发。
> - 导入方式：统一使用 `@kit.*` 命名空间（如 `@kit.ArkData`、`@kit.ArkUI`、`@kit.AbilityKit`）
> - 严格模式：禁止 `any`、`as any`、`Record<string, any>`。使用 `object` 或自定义 interface
> - 状态管理：`@StorageLink` / `@LocalStorageLink` 须配合显式类型标注
> - Canvas2D：API 20 增强了 CanvasRenderingContext2D 的绘制性能
> - 初始化顺序：EntryAbility.onCreate -> 初始化 AppStorage -> 初始化 DB/Config -> loadContent('pages/Index')

## 产品定位与核心价值

### 产品定位
"三问高效学习机"是一款以用户提问为核心触发机制的**智能学习工具**。

- 核心机制：提问 -> 课程生成 -> 三问学习法 -> 深度理解
- 目标用户：大学生、知识工作者、自学者
- 学习范围：任何大学课程或领域知识
- 学习方式：柔性、自主、个性化

### 三大核心价值

| 价值 | 说明 |
|-----|------|
| **按需学习** | 课程内容完全由用户提问触发，避免信息过载 |
| **认知加速** | 通过三问法快速建立框架、挖掘深层、验证理解 |
| **个性化适配** | AI资料 + 用户资料融合，形成专属知识库 |

## 核心业务流程

### 用户学习全生命周期

```
1. 提问阶段
   用户在搜索框输入学习问题
   v
   系统自动识别领域、生成课程框架
   v
   创建课程卡片、进入学习空间

2. 三问学习阶段
   第一问：知识图谱呈现
   +-- 展示学科底层逻辑框架
   |
   第二问：争议点分析
   +-- 对比多源观点、挖掘深层分歧
   |
   第三问：认知测评
   +-- 6个维度自测、评估理解深度

3. 完成与反馈阶段
   生成学习报告、能力雷达图、测评分析
   v
   支持数据导出、课程归档
```

## 核心功能模块详解

### 模块1: 柔性课程生成

**触发机制**
- 用户提问 -> LLM识别领域 -> 自动创建课程 -> 进入学习空间

**课程生命周期状态**

| 状态 | 描述 | UI表现 | 数据库值 |
|-----|------|--------|---------|
| **created** | 首次提问，框架已生成 | 卡片显示，进度0% | 'created' |
| **active** | 正在学习中 | 卡片高亮，进度条更新 | 'active' |
| **completed** | 三问全部完成 | 卡片置灰，显示"已完成" | 'completed' |
| **archived** | 用户归档 | 移至归档列表，不在首页显示 | 'archived' |

**多课程并行支持**
- 用户可创建多个课程（跨学科）
- 课程按最近访问时间排序
- 未激活课程默认隐藏（保持界面简洁）

### 模块2: 复合知识库构建

**AI生成资料**
```
LLM根据课程主题生成结构化文档
  +-- 内容去重（基于标题相似度）
  +-- 支持代码块标记（[CODE]...[/CODE]）
  +-- 与学科关联（按domain过滤）
```

**资料处理流程**
1. LLM生成文档内容，支持可变数量（1-3篇）
2. ContentUtils解析内容段落（区分文本/代码块）
3. 文档与课程关联存储
4. 支持按学科domain筛选

### 模块3: 三问认知引擎

#### 第一问：核心心智模型提取
- **输入**：课程主题 + 学科领域
- **处理**：LLM分析，提取学科底层逻辑框架
- **输出**：知识图谱（节点=核心概念，连线=逻辑关系）
- **组件**：KnowledgeGraphView（Canvas力导向图）
  - CanvasRenderingContext2D 绘制节点和连线
  - 支持点击节点展开/折叠详情
  - `.key()` 强制刷新（解决 @Prop 不触发重绘的API 20坑）

#### 第二问：学术分歧挖掘
- **输入**：AI生成的多角度内容
- **处理**：LLM识别关键争议点、整理正反方证据
- **输出**：争议点卡片（正方观点 vs 反方观点）
- **组件**：ControversyView（左右分栏展示）

#### 第三问：深度测评生成（自由答题模式）

> 设计原则：采用**开放题模式**，而非选择题。用户自由撰写回答，LLM评判。

- **基础**：布鲁姆认知层级（6个维度）
- **维度**：记忆 -> 理解 -> 应用 -> 分析 -> 评价 -> 创造
- **交互流程**：
  1. 系统展示题目（含布鲁姆层级标签）
  2. 用户在文本框中自由输入回答
  3. 调用LLM评判用户回答质量：评分（0-100）+ 针对性评语
  4. 保存完整的对话原始记录
  5. 全部答题完毕后生成评估报告

### 模块4: 学习进度追踪

**可视化元素**
1. **进度条**：整体完成度百分比（CourseHealthService检测）
2. **能力雷达图**：6维度能力评分展示（Canvas绘制）
3. **每周学习时长**：柱状图展示（PersonalCenter）
4. **各领域掌握度**：对比展示

### 模块5: 用户交互与社区

**卡片操作**
- 点击卡片：进入课程详情
- 归档：移入归档列表（ArchiveManagementPage）
- 删除：彻底移除课程及相关数据
- 导出：JSON/Markdown格式导出测评报告

### 模块6: 知识库总览 (KnowledgeHub) — 新增

跨课程所有文档的聚合页面，独立为一个Tab。

**功能：**
- **文档聚合**：展示所有课程下AI生成的文档
- **学科筛选**：根据课程领域(domain)进行过滤
- **关键字搜索**：搜索文档标题和内容
- **DocSheet弹窗**：点击文档标题弹出内容详情
  - 支持代码块高亮（ContentUtils解析[CODE]标记）
  - 支持自然段落分割
  - 底部"去课程"按钮跳转到对应课程详情页

**数据流：**
```
KnowledgeHub.aboutToAppear()
  -> CourseService.getCoursesByUser(userId) 获取所有课程
  -> KnowledgeBaseService.getDocumentsByUser(userId) 获取所有文档
  -> 按domain筛选 + 关键字搜索
  -> 列表展示
  -> 点击文档 -> DocSheet弹窗显示内容
```

### 模块7: UI主题系统 (UiStyle) — 新增

支持3套主题配色 + 深色模式自适应。

**主题列表：**

| 主题名称 | 标签 | 亮色特点 | 深色特点 |
|---------|------|---------|---------|
| default | 默认 | 白色卡片+蓝色主调 | 灰色卡片+蓝色主调 |
| warm | 暖阳 | 米色底色+棕色主调 | 深棕底色+金色主调 |
| midnight | 暗夜 | 白色卡片+金色主调 | 黑色卡片+金色主调 |

**使用方式：**
```typescript
// 在任何页面中
colors(): StyleColors {
  return UiStyle.getColors(this.uiStyle || 'default', this.themeMode === 'dark');
}
```

**AppStorage 存储：**
- 'uiStyle': 'default' | 'warm' | 'midnight'
- 'themeMode': 'auto' | 'light' | 'dark'

### 模块8: 事件总线 (EventBus) — 新增

基于 Map<string, Handler[]> 的全局事件系统，解耦 Service 和 UI。

```typescript
// 支持的事件
const EVENTS = {
  GRAPH_READY: 'graph_ready',
  GRAPH_UPDATED: 'graph_updated',
  CONTROVERSY_READY: 'controversy_ready',
  QUESTIONS_READY: 'questions_ready',
  PROGRESS_UPDATED: 'progress_updated',
  COURSE_CREATED: 'course_created',
  DOCUMENT_ADDED: 'document_added',
  THEME_CHANGED: 'theme_changed'
};
```

### 模块9: DocSheet 文档弹窗

基于 CustomDialog 实现的文档内容查看弹窗，可在 CourseDetailPage 和 KnowledgeHub 中复用。

**特点：**
- 通过 dragBar 拖拽关闭（无自定义 X 按钮）
- ContentUtils.parseContentSegments() 解析文档内容
- 自动检测代码块（[CODE]标记 + 启发式代码行检测）
- 代码块使用等宽字体 + 灰色背景高亮
- 自然段落：按空行/句号分割，每段首行缩进32vp
- tab切换时通过 .key() 强制重建，避免UI闪动

## DocSheet 交互流程

```
用户点击文档标题
  v
DocSheet CustomDialog 弹出
  v
loadDocContent(docId):
  v
ContentUtils.parseContentSegments(doc.content)
  +-- 检查 [CODE]...[/CODE] 标记
  |     -> 标记内的内容作为 code 段
  +-- 无标记时使用启发式检测
  |     -> 缩进行 / 关键字开头的行 / 含特殊符号的行 -> code
  |     -> 其余作为 text (首行缩进32vp)
  v
渲染分段内容 (ForEach + if type判断)
  v
用户拖拽 dragBar 关闭 / 点击"去课程"跳转
```

## UiStyle 使用方式

```typescript
// 在任何页面中定义颜色选择器
import { UiStyle, StyleColors } from '../utils/UiStyle';

@Component
struct MyPage {
  @StorageLink('uiStyle') uiStyle: string = 'default';
  @StorageLink('themeMode') themeMode: string = 'auto';

  colors(): StyleColors {
    return UiStyle.getColors(this.uiStyle || 'default', this.themeMode === 'dark');
  }

  build() {
    Column() {
      Text('内容')
        .fontColor(this.colors().textPrimary);
    }
    .backgroundColor(this.colors().pageBg);
  }
}
```

`StyleColors` 接口：
```typescript
interface StyleColors {
  pageBg: string;          // 页面背景色
  cardBg: string;          // 卡片背景色
  accent: string;          // 主色调
  accentLight: string;     // 主色调浅色变体
  textPrimary: string;     // 主文本色
  textSecondary: string;   // 次级文本色
  textTertiary: string;    // 第三级文本色
  divider: string;         // 分割线色
  cardRadius: number;      // 卡片圆角
  cardShadow: string;      // 卡片阴影(仅warm主题使用)
}
```
