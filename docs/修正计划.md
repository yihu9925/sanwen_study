## P0 — 必须立即修正（影响评分占比最大）

### P0-1：重构第三问测评系统（40%评分占比）

**问题**：当前测评是选择题（A/B/C/D），用户选答案后与预设答案比对判对错。
需求明确要求：
- "自己回答自己产品的学习提问"
- "产品应该对学员的学习效果提出评价和对话原始记录"
- "不要拿AI生成答案喂AI"

**修正方案**：
1. 测评题目改为开放题模式：系统出题（含布鲁姆维度标签），用户自由输入回答
2. 调用LLM评判用户回答：给出评分（0-100）和针对性评语
3. 保存完整的对话原始记录（用户回答 + AI评价 + 时间戳）
4. 测评报告增加：每个题目的用户回答原文、AI评语、分数；按布鲁姆维度汇总
5. 修改 AssessmentService、AssessmentDetailPage、Question 模型

**涉及文件**：
- entry/src/main/ets/models/Question.ets
- entry/src/main/ets/services/AssessmentService.ets
- entry/src/main/ets/services/LLMService.ets
- entry/src/main/ets/pages/AssessmentDetailPage.ets
- entry/src/main/ets/pages/CourseDetailPage.ets
- docs/pages/AssessmentDetailPage.md
- docs/pages/Q3_AssessmentPage.md
- docs/services/AssessmentService.md
- docs/services/LLMService.md
- docs/01_需求与架构.md
- docs/02_数据库设计.md
- docs/04_全局状态管理.md

### P0-2：补充数据库部署脚本

**问题**：数据库建表SQL内联在代码中，没有独立的部署脚本。

**修正方案**：
1. 创建 database/ 目录，存放完整建表SQL脚本
2. SQL脚本包含 course/knowledgebase/document/question/learning_record 五张表
3. 包含索引和外键定义
4. 补充种子数据示例

**涉及文件**：
- database/schema.sql（新建）
- database/seed.sql（新建）
- docs/02_数据库设计.md（更新）

### P0-3：修复 ArkTS API 20 严格模式兼容性

**问题**：代码中存在 Record<string, Object>、as 断言等写法。

**修正方案**：
1. 将 Record<string, Object> 替换为具体 interface
2. 移除不必要的 as 断言
3. 确保所有变量显式类型标注

## P1 — 核心功能完善

### P1-1：创建独立的三问页面文件

**问题**：设计文档规划了 Q1/Q2/Q3 三个独立页面，但实际仅内嵌在 CourseDetailPage Tab 中。

**修正方案**：
1. 创建 pages/Q1_KnowledgeGraphPage.ets
2. 创建 pages/Q2_ControversyPage.ets
3. 创建 pages/Q3_AssessmentPage.ets
4. CourseDetailPage 内跳转到独立页面

### P1-2：实现AI联网资料爬取

**问题**：需求要求自动联网搜索，当前只用了mock数据。

**修正方案**：
1. LLMService 增加 searchWeb 方法
2. CourseService.generateContent 中调用网络搜索补充知识库
3. 确保每个课程有真实的资料内容

### P1-3：实现用户文件上传

**问题**：知识库页面无上传入口。

**修正方案**：
1. 添加文件选择器 UI（@kit.FilePicker）
2. 实现文件解析（至少支持 .txt/.md）
3. 上传后自动加入知识库

### P1-4：补全底部Tab导航

**问题**：Index.ets 只有2个Tab，缺少讨论区。

**修正方案**：
1. Index.ets 添加第三个 TabContent，挂载 DiscussionHub

### P1-5：数据导出功能

**问题**：数据导出完全未实现。

**修正方案**：
1. 实现 JSON/Markdown 导出
2. 在课程详情页和个人中心添加导出入口
