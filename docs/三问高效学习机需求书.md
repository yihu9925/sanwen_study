# 三问高效学习机产品需求书

## 一、产品定位与核心价值

### （一）产品定位

“三问高效学习机”是一款以用户提问为核心触发机制的智能学习工具，通过**柔性课程生成+三问认知引擎**，帮助用户按需构建个性化学习路径，实现从问题提出到深度理解的认知闭环。产品聚焦跨学科学习需求，课程内容完全由用户主动提问触发创建，避免预载信息干扰，保持学习过程的专注性与个性化。

### （二）核心价值

1. **按需学习**：课程内容由用户提问触发生成，真正实现“学什么、建什么”，杜绝信息过载。
2. **认知加速**：通过“三问法”快速建立学科框架、挖掘核心争议、验证理解深度，缩短学习周期。
3. **个性化适配**：课程内容融合用户自主资料与AI智能补充，形成专属复合知识库。
4. **进度可控**：课程状态随学习行为动态变化，仅展示已创建或正在学习的课程，保持界面简洁。

## 二、核心功能需求

### （一）柔性课程生成模块

1. **提问触发课程创建**：用户通过搜索框输入学习问题，系统自动识别领域关键词，即时生成专属课程包并进入学习空间。
2. **课程生命周期管理**

   * 未创建状态：未提问的领域不显示任何课程信息。
   * 已创建状态：首次提问后生成课程卡片，加入主界面展示。
   * 活跃状态：正在学习时卡片高亮，进度条动态更新。
   * 完成状态：“三问法”全部完成后卡片置灰，支持归档或分享。
3. **多课程并行支持**：用户可通过多次提问创建多个课程，已创建课程按最近使用时间排序展示，未激活课程默认隐藏。

### （二）复合知识库构建模块

1. **AI智能资料补充**：课程创建后，系统自动联网检索权威论文、官方文档、行业报告等资料，补充至知识库。
2. **自主资料上传**：用户可上传PDF、Word、Markdown等格式的学习资料，系统自动解析并关联对应课程模块。
3. **资料动态融合**：将AI补充资料与用户上传资料进行语义融合，生成个性化复合知识库，支持关键词搜索与分类筛选。

### （三）三问认知引擎模块

1. **第一问：核心心智模型提取**
分析复合知识库，识别学科底层逻辑框架，以动态知识图谱呈现。知识图谱支持节点展开、逻辑关系查看、关联资料跳转等操作。
2. **第二问：学术分歧挖掘**
对比多源资料观点，自动标注关键争议点及正反方证据，生成“争议-证据-结论”逻辑链图谱。支持争议点筛选、证据详情查看、观点对比分析等功能。
3. **第三问：深度测评生成**
基于布鲁姆认知层级，生成**记忆-理解-应用-分析-评价-创造**六个维度的自测题。支持即时答题、答案解析、错题溯源、测评报告生成等功能。

### （四）学习进度追踪模块

1. **课程进度可视化**：已创建课程以卡片形式展示，包含学科名称、进度条、核心能力雷达图缩略图。
2. **能力维度分析**：自动生成“概念理解、批判思维、实践迁移”等维度的能力雷达图，支持跨学科横向对比。
3. **个性化提醒**：基于学习行为数据，推送任务提醒、争议点深度解读、测评复习建议等内容。

### （五）用户交互模块

1. **课程卡片管理**：支持课程卡片排序、归档、删除等操作，长按卡片可进行资料上传、课程导出等操作。
2. **数据导出功能**：支持导出学习报告、知识图谱图片、测评错题集等内容，格式包含PDF、JSON、Markdown。

## 三、界面设计需求

### （一）主界面（初始状态）

* 布局结构：简洁主屏，仅展示搜索框与引导语。
* 交互逻辑：用户输入问题后，即时生成课程并进入学习空间。
* 视觉风格：采用极简设计，主色调为科技蓝，无冗余信息干扰。

### （二）学习空间界面

1. 心智模型区：顶部展示动态知识图谱，节点为核心概念，连线表示逻辑关系。
2. 争议点区：中部以左右分栏形式呈现争议双方观点，支持切换不同争议主题。
3. 资料区：底部展示复合知识库列表，包含AI补充资料与用户上传资料。

### （三）测评中心界面

1. 测评列表：按认知层级分类展示自测题，标注题目难度与对应知识点。
2. 答题界面：单题展示模式，支持标记题目、查看解析、提交答案等操作。
3. 结果反馈：生成测评报告，包含正确率、能力维度得分、错题解析及复习建议。

### （四）个人中心界面

* 核心功能：包含学习数据总览、课程归档管理、资料上传历史、设置选项等模块。
* 数据可视化：以折线图展示周学习时长变化，以柱状图对比各学科能力得分。

## 四、技术实现要求

1. **架构设计**：采用模块化开发，课程生成引擎与核心认知引擎解耦，支持动态扩展。
2. **NLP能力**：接入大语言模型，实现问题意图识别、知识图谱生成、争议点挖掘、测评题生成等功能。
3. **数据存储**：使用本地数据库实现课程数据持久化，支持资料加密存储与多端数据同步。
4. **性能要求**：课程生成响应时间≤2秒，知识图谱加载时间≤2秒，资料解析准确率≥95%。

## 五、项目进度规划

1. **需求确认阶段**：完成需求评审与界面设计稿确认。
2. **核心开发阶段**：实现柔性课程生成、复合知识库构建、三问认知引擎三大核心模块。
3. **测试优化阶段**：进行功能测试、性能测试与用户体验优化。
4. **上线部署阶段**：完成应用上架与用户培训。
