# 全局状态管理 — AppStorage / LocalStorage

> ⚠ API 20 严格模式下，`@StorageLink` / `@StorageProp` 须显式标注类型

## 状态管理架构

```
┌─────────────────────────────────────────────────────┐
│                    AppStorage                         │
│  (应用级全局状态，所有组件可访问)                     │
│                                                       │
│  userId: string                ← 用户ID               │
│  courseList: Course[]          ← 课程列表             │
│  currentCourseId: string       ← 当前课程ID           │
│  themeMode: 'light'|'dark'     ← 主题模式             │
│  activeTabIndex: number        ← 底部Tab索引          │
└──────────────────────┬──────────────────────────────┘
                       │
          ┌────────────┴────────────┐
          ▼                         ▼
   LocalStorage (页面级)      @StorageLink (组件级)
   ┌──────────────────┐     ┌──────────────────────┐
   │ CourseDetailPage │     │ HomePage             │
   │ pageTitle: string│     │ @StorageLink('userId')│
   │ searchText: str  │     │ @StorageLink('course')│
   └──────────────────┘     └──────────────────────┘
```

## AppStorage 全局状态定义

```typescript
// AppStorageKeys.ts — 统一管理 key 常量（API 20 严格模式）
export class AppStorageKeys {
  static readonly USER_ID = 'userId';
  static readonly COURSE_LIST = 'courseList';
  static readonly CURRENT_COURSE_ID = 'currentCourseId';
  static readonly THEME_MODE = 'themeMode';
  static readonly ACTIVE_TAB_INDEX = 'activeTabIndex';
}

// AppStorage.ets — 初始化（在 App.ets 的 aboutToAppear 中调用）
import { AppStorageKeys } from './AppStorageKeys';

export class AppState {
  static init(): void {
    if (!AppStorage.has(AppStorageKeys.USER_ID)) {
      AppStorage.set<string>(AppStorageKeys.USER_ID, 'default_user');
    }
    if (!AppStorage.has(AppStorageKeys.COURSE_LIST)) {
      AppStorage.set<object[]>(AppStorageKeys.COURSE_LIST, []);
    }
    if (!AppStorage.has(AppStorageKeys.CURRENT_COURSE_ID)) {
      AppStorage.set<string>(AppStorageKeys.CURRENT_COURSE_ID, '');
    }
    if (!AppStorage.has(AppStorageKeys.THEME_MODE)) {
      AppStorage.set<string>(AppStorageKeys.THEME_MODE, 'auto');
    }
    if (!AppStorage.has(AppStorageKeys.ACTIVE_TAB_INDEX)) {
      AppStorage.set<number>(AppStorageKeys.ACTIVE_TAB_INDEX, 0);
    }
  }
}
```

## @StorageLink 组件绑定

```typescript
@Entry
@Component
struct HomePage {
  // API 20 严格模式：类型不可省略
  @StorageLink('userId') userId: string = '';
  @StorageLink('courseList') courseList: object[] = [];
  @StorageLink('currentCourseId') currentCourseId: string = '';
  @StorageLink('themeMode') themeMode: string = 'auto';

  // 页面内状态
  @State searchText: string = '';
  @State isLoading: boolean = false;

  aboutToAppear(): void {
    this.loadCourses();
  }

  loadCourses(): void {
    // 从数据库加载
    const courses = await CourseService.getCourseList(this.userId);
    AppStorage.set<object[]>('courseList', courses);
  }

  selectCourse(courseId: string): void {
    AppStorage.set<string>('currentCourseId', courseId);
    router.pushUrl({ url: 'pages/CourseDetailPage', params: { courseId } });
  }
}
```

## 状态变更流程

```typescript
// 1️⃣ 用户点击课程 → AppStorage 写入
// HomePage.ets
handleCourseClick(courseId: string): void {
  AppStorage.set<string>('currentCourseId', courseId);
  router.pushUrl({ url: 'pages/CourseDetailPage', params: { courseId } });
}

// 2️⃣ CourseDetailPage 读取
// CourseDetailPage.ets
@StorageLink('currentCourseId') currentCourseId: string = '';

aboutToAppear(): void {
  // 从参数或AppStorage获取
  const id = this.currentCourseId || 
    (router.getParams()?.['courseId'] as string) || '';
  this.loadCourse(id);
}

// 3️⃣ 完成学习 → 更新列表
// CourseDetailPage.ets
completeQuestion(qNum: number): void {
  CourseService.updateProgress(this.courseId, qNum);
  // 通知首页刷新
  AppStorage.set<boolean>('needRefresh', true);
}
```

## 状态管理规则

| 状态类型 | 存储位置 | 生命周期 | 访问范围 | 示例 |
|---------|---------|---------|---------|------|
| 全局只读 | AppStorage | 应用全程 | 所有组件 | userId, themeMode |
| 全局读写 | AppStorage | 应用全程 | 所有组件 | courseList, currentCourseId |
| 页面本地 | @State | 页面存在期 | 本组件 | searchText, tabIndex |
| 父子传递 | @Prop / @Link | 组件树 | 父子链 | cardData, onSelect |
| 跨级传递 | @Provide/@Consume | 组件树 | 提供者子树 | theme, courseTitle |

## 状态持久化

```typescript
// 应用启动时从数据库恢复状态
export class StatePersistence {
  static async restoreState(): Promise<void> {
    const prefs = await PreferencesManager.getAll();
    if (prefs.userId) AppStorage.set<string>('userId', prefs.userId as string);
    if (prefs.themeMode) AppStorage.set<string>('themeMode', prefs.themeMode as string);
    // courseList 从 RDB 加载
    const courses = await CourseService.getCourseList(
      AppStorage.get<string>('userId') || ''
    );
    AppStorage.set<object[]>('courseList', courses);
  }

  // 退出时保存关键状态
  static async saveState(): Promise<void> {
    const userId = AppStorage.get<string>('userId');
    const themeMode = AppStorage.get<string>('themeMode');
    if (userId) await PreferencesManager.set('userId', userId);
    if (themeMode) await PreferencesManager.set('themeMode', themeMode);
  }
}
```


## EventBus 与 AppStorage 协同

EventBus 负责"事件通知"，AppStorage 负责"数据共享"。二者配合：

```
Service 完成异步任务
  → EventBus.emit('event_name', data)
  → UI组件监听 EventBus.on('event_name', handler)
  → handler 中更新 @State 或 AppStorage
  → ArkUI 自动触发重新渲染
```

### 推荐的实践模式

```typescript
// 页面级别: 在 aboutToAppear 中订阅，aboutToDisappear 中取消
@Component
struct CourseDetailPage {
  aboutToAppear(): void {
    EventBus.on('graph_ready', this.handleGraphReady);
    EventBus.on('progress_updated', this.handleProgress);
  }

  aboutToDisappear(): void {
    EventBus.off('graph_ready', this.handleGraphReady);
    EventBus.off('progress_updated', this.handleProgress);
  }
}

// 应用级别: 在 App.ets 中订阅，永不取消
// App.ets
@Entry
@Component
struct App {
  aboutToAppear(): void {
    EventBus.on('theme_changed', (data: object) => {
      const mode = (data as Record<string, string>).themeMode;
      AppStorage.set<string>('themeMode', mode);
    });
  }
}
```

## 三种状态管理模式的选型

| 场景 | 推荐方式 | 理由 |
|------|---------|------|
| Tab间共享（userId, courseList） | AppStorage | 全局可见，持久化 |
| 父子组件传递（course对象给card） | @Prop | 单向，简单明确 |
| 跨级组件（themeMode深层传递） | @Provide/@Consume | 避免逐层传递 |
| 兄弟组件（tab切换刷新） | EventBus | 解耦，一对多通知 |
| 页面内临时（搜索词、Loading） | @State | 局部，无需共享 |

### 答题流程状态（自由答题模式）

```typescript
// AppStorage 存储当前答题状态
interface AnswerFlowState {
  courseId: string;
  currentQuestionIndex: number;
  questions: Question[];
  answers: AnswerRecord[];       // 用户已提交的回答
  isEvaluating: boolean;         // 是否在等待LLM评判
  showReport: boolean;           // 是否显示报告
}

interface AnswerRecord {
  questionId: string;
  userAnswerText: string;       // 用户自由输入的答案
  evaluation: AiEvaluation | null;  // LLM评判结果
  answeredAt: string;
}

// 答题流程状态迁移
// 加载题目 → 用户输入 → 提交回答 → LLM评判 → 下一题/报告
// IDLE → ANSWERING → EVALUATING → REVIEWING → NEXT / REPORT
export enum AnswerFlowPhase {
  IDLE,        // 初始状态
  ANSWERING,   // 用户正在输入回答
  EVALUATING,  // LLM正在评判（显示加载）
  REVIEWING,   // 展示评判结果（用户可查看评语）
  FINISHED     // 全部答完，展示报告
}
```

### 状态迁移图

```
IDLE → 加载题目 → ANSWERING
ANSWERING → 用户提交 → EVALUATING
EVALUATING → LLM返回结果 → REVIEWING
REVIEWING → 用户点击"下一题" → ANSWERING（还有更多题）
REVIEWING → 用户点击"查看报告" → FINISHED（最后一题）
FINISHED → 生成报告 → 显示评估报告
```
