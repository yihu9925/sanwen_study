# 全局状态管理 — AppStorage / EventBus

> ⚠ API 20 严格模式下，`@StorageLink` / `@StorageProp` 须显式标注类型

## 状态管理架构

```
+---------------------------------------------------------+
|                     AppStorage                            |
|  (应用级全局状态，所有组件可访问)                        |
|                                                          |
|  userId: string                <- 用户ID                  |
|  themeMode: string             <- 'auto'|'light'|'dark'   |
|  uiStyle: string               <- 'default'|'warm'|'midnight' |
|  activeTabIndex: number        <- 底部Tab索引             |
|  genStep: string               <- 生成步骤状态文字         |
|  genProgress: number           <- 生成进度 0-100          |
+------------------------+----------------------------------+
                         |
           +-------------+---------------+
           |                              |
    @StorageLink (组件级)         EventBus (事件驱动)
    HomePage:                    Service完成后emit事件
      userId, themeMode          UI组件订阅后更新@State
    CourseDetailPage:            -> ArkUI自动重渲染
      themeMode, uiStyle,
      genStep, genProgress,
      activeTabIndex
```

## AppStorage 全局状态变量

| Key | 类型 | 默认值 | 说明 | 使用组件 |
|-----|------|--------|------|---------|
| userId | string | 'default_user' | 当前用户ID | 各Service |
| themeMode | string | 'auto' | 深色模式: 'auto'/'light'/'dark' | Index, 所有页面 |
| uiStyle | string | 'default' | UI主题: 'default'/'warm'/'midnight' | Index, 所有页面 |
| activeTabIndex | number | 0 | 底部Tab索引 | Index |
| genStep | string | '' | 内容生成步骤描述文字 | CourseDetailPage |
| genProgress | number | 0 | 内容生成进度百分比 | CourseDetailPage |

## 初始化（在 EntryAbility 中）

```typescript
// EntryAbility.onCreate 中初始化
async onCreate(want: Want, launchParam: AbilityConstant.LaunchParam): Promise<void> {
  // 1. 初始化全局状态
  if (!AppStorage.has('themeMode')) {
    AppStorage.set('themeMode', 'auto');
  }

  // 2. 应用保存的深色模式
  const savedTheme = AppStorage.get('themeMode') || 'auto';
  if (savedTheme === 'dark') {
    this.context.getApplicationContext().setColorMode(ConfigurationConstant.ColorMode.COLOR_MODE_DARK);
  } else if (savedTheme === 'light') {
    this.context.getApplicationContext().setColorMode(ConfigurationConstant.ColorMode.COLOR_MODE_LIGHT);
  } else {
    this.context.getApplicationContext().setColorMode(ConfigurationConstant.ColorMode.COLOR_MODE_NOT_SET);
  }

  // 3. 初始化数据库和配置
  await RelationalStoreManager.init(this.context);
  await ConfigService.init(this.context);

  // 4. 加载主页面
  this.windowStage.loadContent('pages/Index', (err) => { ... });
}
```

## `@StorageLink` 组件绑定

```typescript
@Entry
@Component
struct CourseDetailPage {
  @StorageLink('themeMode') themeMode: string = 'auto';
  @StorageLink('uiStyle') uiStyle: string = 'default';
  @StorageLink('genStep') generationStep: string = '';
  @StorageLink('genProgress') generationProgress: number = 0;

  aboutToAppear(): void {
    const params = router.getParams() as Record<string, string>;
    this.courseId = params?.['courseId'] || '';
    if (this.courseId) {
      this.loadCourseData();
      this.setupEventListeners();
    }
  }
}
```

## 状态管理规则

| 状态类型 | 存储位置 | 生命周期 | 访问范围 | 示例 |
|---------|---------|---------|---------|------|
| 全局只读 | AppStorage | 应用全程 | 所有组件 | userId, themeMode, uiStyle |
| 全局读写 | AppStorage | 应用全程 | 所有组件 | activeTabIndex, genStep |
| 页面本地 | @State | 页面存在期 | 本组件 | searchText, isLoading |
| 父子传递 | @Prop | 组件树 | 父子链 | 无（当前组件不使用） |
| 跨级传递 | 未使用 | — | — | — |

注：当前项目未使用 @Prop/@Link/@Provide/@Consume 跨级传递模式，父子组件通过对象属性直接传值。

## EventBus 事件系统

EventBus 是全局事件总线（基于 Map<string, Handler[]>），用于解耦 Service 和 UI 组件。

### 支持的8种事件类型

| 事件名称 | 触发时机 | 携带数据 | 监听者 |
|---------|---------|---------|--------|
| graph_ready | Q1知识图谱生成完成 | `{ courseId }` | CourseDetailPage |
| graph_updated | 图谱数据更新 | `{ courseId }` | CourseDetailPage |
| controversy_ready | Q2争议点生成完成 | `{ courseId }` | CourseDetailPage |
| questions_ready | Q3测评题生成完成 | `{ courseId }` | CourseDetailPage |
| progress_updated | 课程进度更新 | `{ courseId, percent }` | CourseDetailPage |
| course_created | 新课程创建 | `{ courseId }` | HomePage |
| document_added | 文档添加 | `{ courseId }` | 资料库 |
| theme_changed | 主题切换 | `{ themeMode }` | 各页面 |

### 使用模式

```typescript
// 1. 定义事件常量
export const EVENTS = {
  GRAPH_READY: 'graph_ready',
  GRAPH_UPDATED: 'graph_updated',
  CONTROVERSY_READY: 'controversy_ready',
  QUESTIONS_READY: 'questions_ready',
  PROGRESS_UPDATED: 'progress_updated',
  COURSE_CREATED: 'course_created',
  DOCUMENT_ADDED: 'document_added',
  THEME_CHANGED: 'theme_changed'
};

// 2. Service 完成任务后 emit
class LLMService {
  static async generateGraph(courseId: string): Promise<void> {
    // ... 异步任务 ...
    EventBus.emit('graph_ready', { courseId });
  }
}

// 3. UI组件在 aboutToAppear 中订阅, aboutToDisappear 取消
@Component
struct CourseDetailPage {
  aboutToAppear(): void {
    this.setupEventListeners();
  }

  setupEventListeners(): void {
    EventBus.on('graph_ready', this.handleGraphReady);
    EventBus.on('progress_updated', this.handleProgress);
  }

  aboutToDisappear(): void {
    EventBus.off('graph_ready', this.handleGraphReady);
    EventBus.off('progress_updated', this.handleProgress);
  }

  handleGraphReady = (data?: object): void => {
    const payload = data as Record<string, string>;
    if (payload.courseId === this.courseId) {
      this.loadGraphData();
    }
  };
}
```

## 状态变更流程

```
Service (异步操作完成)
  |
  +-> EventBus.emit('event_name', data)
  |       |
  |       +-> UI组件中已注册的 EventBus.on 回调执行
  |               |
  |               +-> 更新 @State 变量
  |               |       |
  |               |       +-> ArkUI 自动触发重渲染
  |               |
  |               +-> 更新 AppStorage (全局共享)
  |                       |
  |                       +-> 所有 @StorageLink 组件更新
```

## 状态选型指南

| 场景 | 推荐方式 | 理由 |
|------|---------|------|
| Tab间共享（themeMode, uiStyle） | AppStorage | 全局可见，持久化 |
| Tab切换索引（activeTabIndex） | AppStorage | 跨Tab保持 |
| 页面内临时（搜索词、Loading） | @State | 局部，无需共享 |
| 兄弟组件通知（Tab切换刷新） | .key() + @State | 强制重建组件 |
| Service->UI通知 | EventBus | 解耦，一对多 |
| 生成进度（genStep/genProgress） | AppStorage | 跨页面可见 |

## 课程健康状态（通过 CourseHealthService 检测）

```typescript
export interface CourseContentStatus {
  courseId: string;
  q1KnowledgeGraph: boolean;    // Q1知识图谱是否已生成
  q2Controversies: boolean;     // Q2争议点是否已生成
  q3Questions: boolean;         // Q3测评题是否已生成
  allComplete: boolean;         // 全部完成
  missingParts: string[];       // 缺失的部分名称
  progressPercent: number;      // 综合进度 0-100
}
```

- CourseDetailPage 加载时自动调用 `CourseHealthService.checkCourseStatus(courseId)`
- 检测结果展示在课程详情顶部的健康状态栏
- 缺失部分可通过点击补全按钮单独调用 LLM 生成
- 补全完成后通过 EventBus 重新检测并刷新 UI
