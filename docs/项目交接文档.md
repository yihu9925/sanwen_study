## 二、核心架构

### 页面结构
```
Index (底部Tab 2个)
+-- HomePage (Tab1: 首页)
|   +-- CourseDetailPage (课程详情)
|       +-- Tab1: KnowledgeGraphTab
|       |   +-- KnowledgeGraphView (Canvas)
|       |   +-- RadarChartView (Canvas)
|       |   +-- 章节导航列表
|       +-- Tab2: ControversyTab
|       +-- Tab3: AssessmentTab
|       +-- Tab4: MaterialsTab (资料库)
+-- PersonalCenter (Tab2: 个人中心)
    +-- ArchiveManagementPage (归档)
    +-- SettingsPage (设置+AI检测)
```

### 数据流
```
提问 -> CourseService.createCourse()
  -> LLM识别领域 -> 创建course+knowledgebase记录
  -> CourseDetailPage.loadCourseData()
    -> 加载知识图谱/争议点/题目/文档
    -> 若空则 generateContent():
        1. generateAIDocuments() -> 3篇资料(去重)
        2. 构建 materialContent(资料+问题)
        3. LLM生成知识图谱
        4. LLM生成争议点
        5. LLM生成测评题
    -> 检测健康状态
```

## 六、关键设计决策

- 自由答题模式: TextArea+LLM评判替代选择题 (满足需求不要拿AI生成答案喂AI)
- Canvas .key()重绘: @Prop变化不会触发Canvas onReady, 用key变化强制重建
- LLM中文Prompt: 所有system/user prompt改为中文
- 双Canvas不冲突: 用独立 graphKey/radarKey 分别控制
- 外键级联删除: PRAGMA foreign_keys=ON + ON DELETE CASCADE
- ALTER TABLE迁移: 新列通过try-catch包裹的ALTER TABLE添加
- 健康检测: 检查JSON对象的nodes数组长度而非直接.length
