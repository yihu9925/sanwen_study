# 三问高效学习机 — 系统完善度评估与设计文档更新

## 一、三问系统完善度评估

### 当前架构

```
提问 "什么是微积分？"
  ↓
CourseService.createCourse()
  +-- LLM识别领域 -> "数学"
  +-- 创建 course 记录 + knowledgebase 记录
  +-- 跳转 CourseDetailPage
        ↓
loadCourseData()
  +-- 加载知识图谱、争议点、测评题、资料库
  +-- 若内容为空 -> generateContent()
  │    +-- 1. AI生成3篇资料 -> document表
  │    +-- 2. 加载资料文档 -> 构建materialContent
  │    +-- 3. 生成知识图谱 -> knowledgebase.knowledge_graph
  │    +-- 4. 生成争议点 -> knowledgebase.controversies
  │    +-- 5. 生成测评题 -> question表
  +-- 刷新健康状态 -> contentStatus
```

### 第一问：知识图谱

| 需求项 | 状态 | 评价 |
|--------|------|------|
| 动态知识图谱呈现 | ✅ | Canvas绘制力导向图，节点+关系线 |
| 节点展开/折叠 | - | selectedNode可高亮，无展开折叠交互 |
| 逻辑关系查看 | ✅ | 节点间连线标注关系类型 |
| 关联资料跳转 | ❌ | 点击节点应跳转到对应资料文档 |

**学习有效性**：知识图谱帮助建立学科框架，对视觉学习者效果好。但当前图谱自动布局（固定坐标），非力导向算法，节点过多时会拥挤。章节导航列表是对图谱的补充。

### 第二问：学术分歧挖掘

| 需求项 | 状态 | 评价 |
|--------|------|------|
| 争议点正反方对比 | ✅ | ControversyView左右分栏展示 |
| 证据详情查看 | ✅ | 点击可查看详细证据 |
| 观点对比分析 | - | 基本正反方对比，缺多观点并列对比 |

**学习有效性**：争议点对比是批判性思维训练的核心工具。当前实现能展示正反方观点但缺"证据链"可视化。

### 第三问：深度测评

| 需求项 | 状态 | 评价 |
|--------|------|------|
| 布鲁姆6维度 | ✅ | 生成1-6各层级的题目 |
| 自由答题+LLM评判 | ✅ | 用户输入->LLM评分+评语 |
| 对话原始记录保存 | ✅ | user_answer_text + ai_evaluation |
| 答案解析 | ✅ | 每题显示LLM评语+建议 |
| 错题溯源 | ❌ | 无薄弱知识点自动归类 |
| 评估报告生成 | ✅ | 按维度汇总+雷达图更新 |

**学习有效性**：自由答题模式比选择题更有深度。但缺少"错题溯源"功能。

### 闭环缺失

三问形成一个"问->学->测"的单向流程，缺少"测->补学"的反馈闭环。
