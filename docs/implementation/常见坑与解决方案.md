# 常见坑与解决方案 — API 20 版

> 基于 HarmonyOS 6.0.2 SDK + OpenHarmony API 20 开发时的常见问题

## 1. 导入路径变更 (@ohos.* → @kit.*)

### ❌ 旧方式（API 12-17 兼容但已不推荐）
```typescript
import dataStore from '@ohos.data.relationalStore';
import router from '@ohos.router';
import fileIo from '@ohos.file.fs';
```

### ✅ API 20 推荐方式
```typescript
import { relationalStore } from '@kit.ArkData';
import { router } from '@kit.ArkUI';
import { fileIo } from '@kit.CoreFileKit';
import { common } from '@kit.AbilityKit';
```

> ⚠ **注意**：不要混用 @ohos.* 和 @kit.* 导入同一模块，会引起编译冲突。

## 2. ArkTS 严格模式 — 禁止 any

### ❌ 错误写法（编译不通过）
```typescript
let data: any = getData();                    // ❌ 禁止 any
const result: Record<string, any> = {};       // ❌ 禁止 Record<string, any>
const x = obj as any;                         // ❌ 禁止 as any
```

### ✅ 正确写法
```typescript
let data: object | null = getData();
const result: Record<string, string | number | boolean> = {};
// 或使用interface
interface MyData {
  name: string;
  age: number;
}
const x = obj as MyData;  // 允许类型缩窄，不允许扩大到 any
```

## 3. 组件状态变量 — 必须标注类型

### ❌ 错误
```typescript
@State data = [];        // ❌ 无法推断类型
@State items = new Map(); // ❌ Map 不能用于 @State
```

### ✅ 正确
```typescript
@State data: Course[] = [];
@State items: string[] = [];  // 使用数组替代 Map
// 或使用 Record
@State itemMap: Record<string, string> = {};
```

## 4. AppStorage 类型必须一致

```typescript
// ✅ 正确：set/get 类型一致
AppStorage.set<string>('userId', 'user_001');
const uid: string = AppStorage.get<string>('userId') || '';

// @StorageLink 类型必须与 AppStorage.set 一致
@StorageLink('userId') userId: string = '';
```

## 5. router.getParams() 返回值处理

```typescript
// API 20 中 router.getParams() 返回 object | undefined
// ✅ 安全获取
const courseId = (router.getParams()?.['courseId'] as string) || '';

// 或用 ! 断言（确定有值）
const courseId: string = router.getParams()!['courseId'] as string;
```

## 6. Canvas 绘制 — 坐标系和像素比

```typescript
// API 20 中 Canvas 坐标单位为 vp（虚拟像素），无需手动缩放
// ❌ 不要这样做
ctx.scale(2, 2);  // 可能导致模糊

// ✅ 直接在 vp 坐标系绘制
ctx.fillStyle = '#2563EB';
ctx.arc(100, 100, 20, 0, Math.PI * 2);  // 100vp, 100vp
```

## 7. 异步函数中的 try-catch

```typescript
// ✅ Service方法必须统一错误处理
async function loadData(): Promise<void> {
  try {
    const result = await riskyOperation();
    // 处理结果
  } catch (error) {
    console.error('Operation failed:', JSON.stringify(error));
    // 显示错误给用户
    promptAction.showToast({ message: '加载失败，请重试' });
  }
}
```

## 8. Preferences 存储限制

- `@kit.ArkData` 的 Preferences 单条 value 最多 **8KB**
- 不要用来存储大数据（如知识图谱JSON）
- 大数据存 RelationalStore，Preferences 存简单键值

## 9. 文件路径问题

```typescript
// API 20 文件路径使用沙箱路径
// ✅ 获取正确路径
import { common } from '@kit.AbilityKit';

const context = getContext() as common.UIAbilityContext;
const cacheDir = context.cacheDir;       // 缓存目录
const filesDir = context.filesDir;       // 文件目录
const tempDir = context.tempDir;         // 临时目录

// 不要在路径中硬编码
// ❌ const path = '/data/storage/el2/base/xxx';
```

## 10. 深色模式适配

```typescript
// ✅ 使用系统色值自动适配
Text('标题')
  .fontColor($r('sys.color.ohos_id_color_text_primary'))  // 自动切换深浅色
  .backgroundColor($r('sys.color.ohos_id_color_background'));

// ❌ 不要硬编码颜色
// .fontColor('#1F2937')  // 深色模式下不可见
```

## 11. 网络请求权限

```typescript
// module.json5 中必须声明
{
  "module": {
    "requestPermissions": [
      { "name": "ohos.permission.INTERNET" },        // 网络
      { "name": "ohos.permission.READ_MEDIA" },      // 读媒体
      { "name": "ohos.permission.WRITE_MEDIA" },     // 写媒体
      { "name": "ohos.permission.STORAGE_MANAGER" }  // 文件管理
    ]
  }
}
```

## 12. API 20 特有：@Reusable 组件

```typescript
// API 20 中可声明可复用组件提高性能
@Reusable
@Component
export struct CourseCard {
  @Prop course: Course;

  aboutToReuse(params: Record<string, object>): void {
    this.course = params.course as Course;
  }
}

// 在 List/Grid 中使用
LazyForEach(this.courses, (course: Course) => {
  CourseCard({ course: course })
    .reuseId('course_card')
}, (course: Course) => course.id)
```
