# 鸿蒙"三问高效学习机"APP — 项目总览

## 项目基本信息

| 项目名称 | 三问高效学习机 |
|---------|---------------|
| 开发语言 | ArkTS + ArkUI |
| 最低API版本 | 20（OpenHarmony API 20） |
| 支持系统 | HarmonyOS 6.0+ |
| SDK版本 | HarmonyOS 6.0.2 SDK |
| 开发模式 | 单应用（支持多端适配） |
| 目标设备 | 手机、平板 |
| 开发周期 | 3-4周 |

## 环境要求

- **OS**: HarmonyOS 6.0+ / OpenHarmony
- **SDK**: HarmonyOS 6.0.2 SDK（内置）
- **API Level**: 20
- **IDE**: DevEco Studio NEXT（或更高版本）
- **语言**: ArkTS（严格模式 — 禁止 any、Record<string, any>）
- **UI框架**: ArkUI（@Component）
- **模块导入方式**: `@kit.*` 命名空间（API 18+ 推荐方式）

> ⚠ 注意：API 20 启用了 ArkTS 严格类型检查，所有变量必须有显式类型，禁止使用 `any`、`as any`、`Record<string, any>`。请使用 `object`、`Record<string, string>` 或自定义 interface 替代。

## 核心价值主张

该产品采用**三问法**（三个显性步骤）帮助学生高效学习任何大学课程或领域知识：

1. **第一问：核心心智模型提取** → 知识图谱展示
2. **第二问：学术分歧挖掘** → 争议点分析
3. **第三问：深度测评生成** → 认知评估

## 核心功能模块

| 模块 | 核心职责 | 关键功能 | 优先级 |
|-----|--------|--------|------|
| **柔性课程生成** | 触发式课程创建 | 提问识别、AI资料生成、课程初始化、多课程并行 | P0 |
| **复合知识库** | AI+用户资料融合 | AI资料生成(去重)、资料上传UI、关键词搜索 | P0 |
| **三问认知引擎** | 学习路径呈现 | 知识图谱Canvas、争议点对比、自由答题+LLM评判 | P0 |
| **知识库总览** | 跨课程文档聚合 | 全课程文档一览、学科筛选、关键字搜索、DocSheet弹窗 | P0 |
| **课程健康检测** | 内容完整性 | 自动检测Q1/Q2/Q3缺失、逐个补全按钮 | P0 |
| **进度追踪** | 数据可视化 | 进度条、布鲁姆6维雷达图、章节导航 | P1 |
| **交互管理** | 用户操作支持 | 卡片归档/删除、知识库总览、数据导出JSON/MD | P1 |
| **个人中心** | 数据总览 | 归档管理、设置、外观主题、AI连通性检测 | P1 |
| **UI主题系统** | 外观个性化 | 多套主题(default/warm/midnight)、深色模式自适应 | P2 |

## 核心概念与术语

| 术语 | 定义 |
|-----|------|
| **课程** | 用户每次提问触发生成的学习内容单位 |
| **知识图谱** | 课程核心概念及其逻辑关系的可视化表示（Canvas力导向图） |
| **争议点** | 学科领域内具有对立观点的学术问题（左右分栏展示） |
| **能力维度** | 根据布鲁姆分类法的6个认知等级（记忆->理解->应用->分析->评价->创造） |
| **知识库** | 课程内AI生成的资料文档集合 |
| **知识库总览** | 跨课程所有文档的聚合页（KnowledgeHub），支持学科筛选+关键字搜索 |
| **DocSheet** | 文档内容弹窗（CustomDialog），支持代码块高亮、自然段落分割 |
| **生命周期状态** | created(已创建) / active(进行中) / completed(已完成) / archived(已归档) |
| **EventBus** | 全局事件总线（Map-based），解耦Service->UI通信 |
| **UiStyle** | 主题配色系统，支持default/warm/midnight三套主题+深色模式 |
