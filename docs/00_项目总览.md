# 鸿蒙"三问高效学习机"APP — 项目总览

## 项目基本信息

| 项目名称 | 三问高效学习机 |
|---------|---------------|
| 开发语言 | ArkTS + ArkUI |
| 最低API版本 | 20（OpenHarmony API 20） |
| 支持系统 | HarmonyOS 6.0+ |
| SDK版本 | HarmonyOS 6.0.2 SDK |
| 开发模式 | 单应用（支持多端适配） |
| 目标设备 | 手机、平板 |
| 开发周期 | 3-4周 |

## 环境要求

- **OS**: HarmonyOS 6.0+ / OpenHarmony
- **SDK**: HarmonyOS 6.0.2 SDK（内置）
- **API Level**: 20
- **IDE**: DevEco Studio NEXT（或更高版本）
- **语言**: ArkTS（严格模式 — 禁止 any、Record<string, any>）
- **UI框架**: ArkUI（@Component / @ComponentV2）
- **模块导入方式**: `@kit.*` 命名空间（API 18+ 推荐方式）

> ⚠ 注意：API 20 启用了 ArkTS 严格类型检查，所有变量必须有显式类型，禁止使用 `any`、`as any`、`Record<string, any>`。请使用 `object`、`Record<string, string>` 或自定义 interface 替代。

## 核心价值主张

该产品采用**三问法**（三个显性步骤）帮助学生高效学习任何大学课程或领域知识：

1. **第一问：核心心智模型提取** → 知识图谱展示
2. **第二问：学术分歧挖掘** → 争议点分析  
3. **第三问：深度测评生成** → 认知评估

## 核心功能模块

| 模块 | 核心职责 | 关键功能 | 优先级 |
|-----|--------|--------|------|
| **柔性课程生成** | 触发式课程创建 | 提问识别、AI资料生成、课程初始化、多课程并行 | P0 |
| **复合知识库** | AI+用户资料融合 | AI资料生成(去重)、资料上传UI、关键词搜索 | P0 |
| **三问认知引擎** | 学习路径呈现 | 知识图谱Canvas、争议点对比、自由答题+LLM评判 | P0 |
| **课程健康检测** | 内容完整性 | 自动检测Q1/Q2/Q3缺失、逐个补全按钮 | P0 |
| **进度追踪** | 数据可视化 | 进度条、布鲁姆6维雷达图、章节导航 | P1 |
| **交互管理** | 用户操作支持 | 卡片归档/删除、讨论区、数据导出JSON/MD | P1 |
| **个人中心** | 数据总览 | 归档管理、设置、AI连通性检测 | P1 |

## 核心概念与术语

| 术语 | 定义 |
|-----|------|
| **课程** | 用户每次提问触发生成的学习内容单位 |
| **知识图谱** | 课程核心概念及其逻辑关系的可视化表示 |
| **争议点** | 学科领域内具有对立观点的学术问题 |
| **能力维度** | 根据布鲁姆分类法的6个认知等级 |
| **复合知识库** | AI爬取资料 + 用户上传资料的融合集合 |
| **生命周期状态** | created(已创建) / active(进行中) / completed(已完成) / archived(已归档) |

## 非功能需求

| 类别 | 要求 | 实现方案 |
|-----|------|--------|
| 性能 | 课程生成≤2s、知识图谱加载≤2s | 异步任务、缓存策略 |
| 适配 | 600vp-1600vp宽度、深色模式 | ResponsiveContainer、@Provider/@Consume 主题 |
| 兼容性 | HarmonyOS 6.0+、API 20 | 使用 @kit.* 标准组件、严格ArkTS |
| 权限 | 网络、文件读写、存储 | module.json5配置、运行时请求 |
| 数据安全 | 本地加密、网络HTTPS | 敏感数据AES加密、接口签名 |
